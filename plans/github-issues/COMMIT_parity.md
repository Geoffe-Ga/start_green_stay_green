docs(lang): add gap-driven issue set for full language parity

Add the GitHub issue bodies and generator for the language-support parity
initiative: bring every supported language to parity across all generators
and add the smartwatch-platform languages (Swift/watchOS, Kotlin & Java/
Wear OS, C/C++ Tizen).

Structure: one umbrella parity epic, gap-driven per-language epics with
component sub-issues (foundation, quality tooling, CI, tests, docs), and
single architecture-parity issues for go/rust. Bodies are generated by
_generate_smartwatch_issues.py from the per-language tooling matrix and the
audited generator gap profiles, so the set is reproducible.
