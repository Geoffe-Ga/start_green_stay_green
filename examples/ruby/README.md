# Ruby Example: `wrist-cadence`

A minimal example of the Ruby project scaffold generated by:

```bash
start-green-stay-green init \
  --project-name wrist-cadence \
  --language ruby \
  --no-interactive
```

## Provenance

Every file under [`wrist-cadence/`](wrist-cadence/) is **verbatim
generated output**: the command above was run against this repository
(with the AI enhancement pass disabled, i.e. the deterministic Pass-1
scaffold) and the Ruby-specific files were copied here unmodified.
Re-running the command reproduces them byte-for-byte.

## What this example includes

| File | Generated by |
|------|--------------|
| `Gemfile` | The dependency manifest with every quality gem pinned (rspec, simplecov, rubocop, bundler-audit, packwerk) — `bundle install` provisions the whole toolchain; the comments record why Brakeman is deliberately absent (Rails-specific; errors on plain-Ruby projects) |
| `.rubocop.yml` | The single home of the whole RuboCop policy — format (Layout), lint (Lint/Style), the `Metrics/CyclomaticComplexity` ≤10 ceiling, and the Security cop department — shared by the pre-commit hook, `scripts/format.sh`, `scripts/lint.sh`, and CI (`TargetRubyVersion: 3.3`, `NewCops: enable`) |
| `lib/wrist_cadence.rb` | Hello-world entry point declaring the `WristCadence` module (PascalCase derived from the project name — the same constant the spec describes) |
| `spec/spec_helper.rb` | RSpec configuration plus the SimpleCov ≥90% line-coverage bound — its single home, activated by `COVERAGE=true` so a plain `rspec` run stays fast; SimpleCov's at_exit hook fails the run directly when the bound is missed |
| `spec/wrist_cadence_spec.rb` | RSpec starter suite exercising the module's behavior (output assertion, not just no-raise) |
| `.pre-commit-config.yaml` | Hooks: RuboCop in check mode from the official rubocop/rubocop manifest (the upstream default args include `--autocorrect` — fixing mode, which can never fail on correctable offenses — so the generated hook overrides args to `--force-exclusion` alone), gitleaks, detect-secrets, shellcheck, hygiene checks |
| `.github/workflows/ci.yml` | ubuntu-runner CI with a Ruby 3.3/3.4 matrix (the maintained lines): gems installed via `ruby/setup-ruby`'s bundler-cache, the full RuboCop run against the same `.rubocop.yml`, bundler-audit (a hard gate in CI, where network access is guaranteed), RSpec with the spec_helper-backed SimpleCov ≥90% gate, and a best-effort Codecov upload. No packaging job: the scaffold ships no `.gemspec` |

RuboCop is one tool wearing four hats — formatter, linter, complexity
gate, and source-level security — and `.rubocop.yml` is the single home
of that policy, so the pre-commit hook, the scripts, and CI cannot
drift from each other.

## What the full generation also produces (omitted here for brevity)

The real run creates 82 files. Omitted from this example because they
are language-independent and identical to the canonical sources in this
repository (`reference/subagents/`, `.claude/skills/`, templates):

- `.claude/agents/` — 8 AI subagent profiles
- `.claude/docs/` + `CLAUDE.md` — modular AI project context
- `.claude/skills/` — 23 Claude Code skills
- `.github/workflows/code-review.yml` — AI code-review workflow
- `README.md` — project readme
- `plans/architecture/` — Packwerk architecture enforcement
  (`packwerk.yml` + `package.yml`, parked as templates plus
  `run-check.sh`; copy both YAML files to the project root and run
  `bundle exec packwerk check` to activate — the packwerk gem is
  already pinned in the Gemfile. The configs document the enforcement
  limits honestly: static constant-reference analysis, the
  Zeitwerk-autoload assumption that keeps the check vacuous on this
  flat plain-Ruby scaffold until packages are defined, and why the
  gate matters — Ruby's `require` does not reject circular requires
  at load time, so `packwerk validate`'s acyclic package graph is the
  only early signal)
- `.secrets.baseline` — the detect-secrets baseline referenced by the
  pre-commit config; not generated by the tool (a known generator gap)
  and omitted here. Create it before the first commit with:
  `detect-secrets scan > .secrets.baseline`
- `scripts/` — `check-all.sh`, `format.sh` (the `--autocorrect`
  fixing path), `lint.sh` (the full RuboCop cop set), `test.sh`
  (≥90% line coverage via SimpleCov with `--coverage`), `security.sh`
  (bundler-audit dependency CVE scan; offline runs warn and skip),
  `pr-status.sh`

## Prerequisites

- Ruby 3.3+ (`brew install ruby` on macOS,
  `apt-get install ruby-full` on Debian/Ubuntu) and Bundler —
  `bundle install` provisions every pinned quality gem, so there is
  nothing else to install
- Network access for the `scripts/security.sh` dependency CVE scan
  (bundler-audit fetches the ruby-advisory-db; offline runs warn and
  skip)

## Getting started with a generated project

```bash
bundle install
bundle exec rspec
pre-commit install
./scripts/check-all.sh
```
