# C# (.NET) Example: `wrist-ledger`

A minimal example of the C# (.NET 8) project scaffold generated by:

```bash
start-green-stay-green init \
  --project-name wrist-ledger \
  --language csharp \
  --no-interactive
```

## Provenance

Every file under [`wrist-ledger/`](wrist-ledger/) is **verbatim generated
output**: the command above was run against this repository (with the AI
enhancement pass disabled, i.e. the deterministic Pass-1 scaffold) and
the C#-specific files were copied here unmodified. Re-running the
command reproduces them byte-for-byte.

## What this example includes

| File | Generated by |
|------|--------------|
| `wrist-ledger.csproj` | The single SDK-style project file that owns the whole quality policy: the Roslyn analyzer configuration (`EnableNETAnalyzers`, `AnalysisLevel` latest, `TreatWarningsAsErrors`), the Coverlet ≥90% line-coverage bound (`Threshold`/`ThresholdType`/`ThresholdStat` — its single home), the `CodeMetricsConfig.txt` AdditionalFiles wiring behind the CA1502 complexity ceiling, and the pinned xUnit / coverlet.msbuild / NetArchTest.Rules / SecurityCodeScan packages |
| `.editorconfig` | The switch that enables the (default-off) CA1502 cyclomatic-complexity rule as an error; `dotnet format` reads the same file |
| `CodeMetricsConfig.txt` | Companion analyzer config — the single home of the CA1502 ≤10 complexity bound (10 reports 11+), wired into every build via the csproj's AdditionalFiles entry |
| `src/Program.cs` | Hello-world entry point in the `WristLedger` namespace (derived from the project name; C# namespaces carry no directory-matching requirement) |
| `tests/MainTests.cs` | xUnit starter suite — src/ and tests/ compile into one assembly, so `dotnet test` runs the whole project |
| `.pre-commit-config.yaml` | Hooks: dotnet format in check mode (`--verify-no-changes`), Roslyn analyzers as a plain `dotnet build --nologo` (the csproj already treats warnings as errors), gitleaks, detect-secrets, shellcheck, hygiene checks (including check-xml for the csproj) |
| `.github/workflows/ci.yml` | ubuntu-runner CI: a .NET 8 SDK quality job (the csproj targets net8.0, so the matrix pins the 8.0 line) running restore, the analyzer-gated build, the dotnet format check, `dotnet test /p:CollectCoverage=true` with the csproj-backed ≥90% Coverlet gate, and a best-effort Codecov upload, plus a build-and-publish job |

Every quality gate is a thin dotnet CLI invocation: the csproj is the
single source of the analyzer and coverage policy, so the scripts,
pre-commit hooks, and CI cannot version-drift from the build.

## What the full generation also produces (omitted here for brevity)

The real run creates 81 files. Omitted from this example because they
are language-independent and identical to the canonical sources in this
repository (`reference/subagents/`, `.claude/skills/`, templates):

- `.claude/agents/` — 8 AI subagent profiles
- `.claude/docs/` + `CLAUDE.md` — modular AI project context
- `.claude/skills/` — 23 Claude Code skills
- `.github/workflows/code-review.yml` — AI code-review workflow
- `README.md` — project readme
- `plans/architecture/` — NetArchTest architecture enforcement
  (`ArchitectureTest.cs`, parked as a template plus `run-check.sh`;
  copy it flat into `tests/` to enforce — C# namespaces need no
  directory match, and NetArchTest.Rules is already declared in the
  csproj so it compiles the moment it lands)
- `.secrets.baseline` — the detect-secrets baseline referenced by the
  pre-commit config; not generated by the tool (a known generator gap)
  and omitted here. Create it before the first commit with:
  `detect-secrets scan > .secrets.baseline`
- `scripts/` — `check-all.sh`, `format.sh`, `lint.sh`, `test.sh`
  (≥90% line coverage via Coverlet), `security.sh` (vulnerable NuGet
  package scan via `dotnet list package --vulnerable`), `pr-status.sh`

## Prerequisites

- The .NET 8 SDK — `brew install dotnet-sdk` (macOS) or
  `apt-get install dotnet-sdk-8.0` (Debian/Ubuntu). `dotnet format`,
  the Roslyn analyzers, and NuGet all ship inside the SDK, so there is
  nothing else to install
- Network access for the `scripts/security.sh` vulnerable-package
  scan (it queries the NuGet vulnerability database; offline runs
  warn and skip)

## Getting started with a generated project

```bash
dotnet test
pre-commit install
./scripts/check-all.sh
```
