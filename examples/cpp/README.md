# C/C++ (Tizen) Example: `wrist-pulse`

A minimal example of the C/C++ Tizen native (Samsung Galaxy Watch)
project scaffold generated by:

```bash
start-green-stay-green init \
  --project-name wrist-pulse \
  --language cpp \
  --no-interactive
```

## Provenance

Every file under [`wrist-pulse/`](wrist-pulse/) is **verbatim generated
output**: the command above was run against this repository (with the AI
enhancement pass disabled, i.e. the deterministic Pass-1 scaffold) and
the C/C++-specific files were copied here unmodified. Re-running the
command reproduces them byte-for-byte.

## What this example includes

| File | Generated by |
|------|--------------|
| `CMakeLists.txt` | CMake ≥3.20 build of ONLY the pure-logic library + Catch2 tests (C++17, `CMAKE_CXX_STANDARD_REQUIRED`, extensions off, `ENABLE_COVERAGE` gcov option) |
| `conanfile.txt` | Conan 2 manifest — Catch2 3.8.0 plus the CMakeDeps/CMakeToolchain generators |
| `tizen-manifest.xml` | Wearable-profile watch-application manifest (app ID `org.example.wristpulse`) |
| `inc/greeting.h` / `src/greeting.cpp` | Pure logic (`format_greeting()`), testable without Tizen Studio |
| `src/main.cpp` | Tizen `watch_app` + EFL entry point — needs the Tizen native SDK, deliberately outside the CMake build |
| `tests/test_greeting.cpp` | Catch2 starter suite |
| `res/README.md` / `shared/res/README.md` | Tizen resource-layout notes (the launcher-icon PNG is a binary artifact and is never scaffolded) |
| `.clang-format` | Google-based formatting config shared by `scripts/format.sh` and the pre-commit hook |
| `.clang-tidy` | Static-analysis config — bugprone/cert/clang-analyzer checks promoted to errors |
| `.pre-commit-config.yaml` | Hooks: clang-format (mirrors-clang-format v18.1.8), clang-tidy, cppcheck, gitleaks, detect-secrets, shellcheck, hygiene checks |
| `.github/workflows/ci.yml` | ubuntu-24.04 CI: quality job running the generated scripts (format check, clang-tidy + cppcheck + lizard, gitleaks, ≥90% lcov coverage gate, flawfinder), plus a gcc/clang build-and-test matrix |

The `.tpk` wearable package is **deliberately out of scope** for both
the CMake build and CI: it is produced by the Tizen Studio CLI
(`tizen build-native` / `tizen package`), and Tizen Studio is a manual
GUI installer that neither the scaffold nor a CI runner can provision.

## What the full generation also produces (omitted here for brevity)

The real run creates 87 files. Omitted from this example because they
are language-independent and identical to the canonical sources in this
repository (`reference/subagents/`, `.claude/skills/`, templates):

- `.claude/agents/` — 8 AI subagent profiles
- `.claude/docs/` + `CLAUDE.md` — modular AI project context
- `.claude/skills/` — 23 Claude Code skills
- `.github/workflows/code-review.yml` — AI code-review workflow
- `README.md` — project readme
- `plans/architecture/` — include-boundary architecture enforcement
  (`check_architecture.py`, a stdlib-only Python checker, plus
  `run-check.sh`)
- `scripts/` — `check-all.sh`, `format.sh`, `lint.sh`, `test.sh`
  (≥90% line coverage via gcov/lcov), `security.sh` (flawfinder
  dangerous-API scan), `pr-status.sh`

## Prerequisites

- CMake ≥3.20 and Conan 2 (the unit tests need nothing else)
- A C++17 compiler — CI exercises both gcc and clang
- `brew install clang-format llvm cppcheck lcov` (macOS; clang-tidy
  ships in the keg-only `llvm` formula) or
  `apt-get install clang-format clang-tidy cppcheck lcov`
- `pip install lizard flawfinder` for the complexity and security gates
- Tizen Studio CLI — **only** for packaging the `.tpk` watch app
  (https://developer.tizen.org/development/tizen-studio)

## Getting started with a generated project

```bash
conan install . --output-folder=build --build=missing
cmake -B build -S . \
    -DCMAKE_TOOLCHAIN_FILE=build/conan_toolchain.cmake \
    -DCMAKE_BUILD_TYPE=Release
cmake --build build
ctest --test-dir build
pre-commit install
./scripts/check-all.sh
```

See [docs/CLI_REFERENCE.md](../../docs/CLI_REFERENCE.md) for the full
C/C++ toolchain table.
