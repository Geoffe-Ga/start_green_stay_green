# Swift (watchOS) Example: `wrist-timer`

A minimal example of the Swift project scaffold generated by:

```bash
start-green-stay-green init \
  --project-name wrist-timer \
  --language swift \
  --no-interactive
```

## Provenance

Every file under [`wrist-timer/`](wrist-timer/) is **verbatim generated
output**: the command above was run against this repository (with the AI
enhancement pass disabled, i.e. the deterministic Pass-1 scaffold) and
the Swift-specific files were copied here unmodified. Re-running the
command reproduces them byte-for-byte.

## What the full generation also produces (omitted here for brevity)

The real run creates 81 files. Omitted from this example because they
are language-independent and identical to the canonical sources in this
repository (`reference/subagents/`, `.claude/skills/`, templates):

- `.claude/agents/` — 8 AI subagent profiles
- `.claude/docs/` + `CLAUDE.md` — modular AI project context
- `.claude/skills/` — 23 Claude Code skills
- `.github/workflows/code-review.yml` — AI code-review workflow
- `README.md` — project readme
- `plans/architecture/` — SwiftLint custom-rule architecture enforcement
  (`.swiftlint-architecture.yml`, `run-check.sh`)
- `scripts/` — `check-all.sh`, `format.sh`, `lint.sh`, `test.sh`
  (≥90% coverage via llvm-cov), `security.sh` (Periphery dead-code scan),
  `pr-status.sh`

## Prerequisites

- Swift 5.9, 5.10, or 6.0 with Swift Package Manager (SPM)
- `brew install swiftlint swift-format gitleaks` for the pre-commit hooks
- `brew install periphery` for `scripts/security.sh` dead-code scans

## Getting started with a generated project

```bash
swift package resolve
swift build
pre-commit install
./scripts/check-all.sh
```

See [docs/CLI_REFERENCE.md](../../docs/CLI_REFERENCE.md) for the full
Swift toolchain table.
