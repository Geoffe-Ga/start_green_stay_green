# Java (Wear OS legacy) Example: `wrist-tempo`

A minimal example of the Java legacy Android Wear (Wear OS) project
scaffold generated by:

```bash
start-green-stay-green init \
  --project-name wrist-tempo \
  --language java \
  --no-interactive
```

## Provenance

Every file under [`wrist-tempo/`](wrist-tempo/) is **verbatim generated
output**: the command above was run against this repository (with the AI
enhancement pass disabled, i.e. the deterministic Pass-1 scaffold) and
the Java-specific files were copied here unmodified. Re-running the
command reproduces them byte-for-byte.

## What this example includes

| File | Generated by |
|------|--------------|
| `pom.xml` | Maven build of ONLY the pure-logic sources (Java 17, JUnit 4 via Surefire, the JaCoCo ≥90% line-coverage bound, and the Checkstyle/PMD/SpotBugs/dependency-check plugins every quality gate invokes) |
| `pmd-ruleset.xml` | Companion PMD ruleset — the single home of the cyclomatic-complexity ≤10 gate, shared by `scripts/lint.sh`, the pre-commit PMD hook, and CI |
| `src/main/java/com/example/wrist_tempo/Greeting.java` | Pure logic (`Greeting.greet()`), testable without the Android SDK |
| `src/test/java/com/example/wrist_tempo/GreetingTest.java` | JUnit 4 starter suite |
| `app/src/main/AndroidManifest.xml` | Legacy Android Wear manifest — watch `uses-feature`, wearable shared library, standalone metadata |
| `app/src/main/java/com/example/wrist_tempo/MainActivity.java` | Watch-app entry point rendering the `BoxInsetLayout` layout — needs the Android SDK, deliberately outside the Maven build |
| `app/src/main/res/layout/activity_main.xml` | `androidx.wear.widget.BoxInsetLayout` layout for round watch faces |
| `.pre-commit-config.yaml` | Hooks: google-java-format, Checkstyle/PMD/SpotBugs as Maven goals, gitleaks, detect-secrets, shellcheck, hygiene checks |
| `.github/workflows/ci.yml` | ubuntu-runner CI: a JDK 17/21 (Temurin) quality matrix running the same Maven goals as the local build (Checkstyle, PMD, compile-first SpotBugs, tests + JaCoCo with the pom-backed `mvn jacoco:check` ≥90% gate, best-effort Codecov upload), plus a build-and-package job |

The watch APK is **deliberately out of scope** for both the Maven build
and CI: `app/` needs the Android SDK and the androidx.wear AAR, which
plain Maven cannot consume (the android-maven-plugin is unmaintained).
Build the app with Android tooling (Android Studio / Gradle), adding
`src/main/java/` as a source root — that Gradle build is not scaffolded.

## What the full generation also produces (omitted here for brevity)

The real run creates 83 files. Omitted from this example because they
are language-independent and identical to the canonical sources in this
repository (`reference/subagents/`, `.claude/skills/`, templates):

- `.claude/agents/` — 8 AI subagent profiles
- `.claude/docs/` + `CLAUDE.md` — modular AI project context
- `.claude/skills/` — 23 Claude Code skills
- `.github/workflows/code-review.yml` — AI code-review workflow
- `README.md` — project readme
- `plans/architecture/` — ArchUnit architecture enforcement
  (`ArchitectureTest.java`, parked as a template plus `run-check.sh`;
  copy it into `src/test/java/` to enforce — ArchUnit is already
  test-scoped in the pom so it compiles the moment it lands)
- `.secrets.baseline` — the detect-secrets baseline referenced by the
  pre-commit config; not generated by the tool (a known generator gap)
  and omitted here. Create it before the first commit with:
  `detect-secrets scan > .secrets.baseline`
- `scripts/` — `check-all.sh`, `format.sh`, `lint.sh`, `test.sh`
  (≥90% line coverage via JaCoCo), `security.sh` (SpotBugs + OWASP
  dependency-check CVE scan), `pr-status.sh`

## Prerequisites

- JDK 17+ (CI also exercises JDK 21) and Maven
- `brew install google-java-format` for the pre-commit format hook
  (the linters are Maven plugins pinned in the pom — no extra installs)
- A free NVD API key (exported as `NVD_API_KEY`) for the OWASP
  dependency-check CVE scan — without one the scan is skipped with a
  warning
- An Android SDK for building the watch APK (Android Studio or
  command-line tools) — **only** for the app module; everything else
  runs without it

## Getting started with a generated project

```bash
mvn test
pre-commit install
./scripts/check-all.sh
```
